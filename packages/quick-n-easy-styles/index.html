Welcome to Quick 'n Easy Styles!

This is a demonstration of a super simple, variable-driven CSS library. The core idea is to allow for significant visual changes to a website by modifying just a handful of CSS root variables. Explore the "Style Preset" dropdown below to see how different aesthetics like Neo-Brutalism, Minimalism, Flat Design, Material-inspired, or a clean Dark Mode can be achieved with minimal effort.

Core CSS Variables

The entire look and feel of this library is controlled by the following 10 primary CSS variables defined in the :root selector of quick-n-easy-styles.css. You can override these in your own stylesheet or directly in the file to customize your site.

  • --qne-bg-color: Sets the main background color of the page. (e.g., #ffffff, #121212)
  • --qne-text-color: Defines the primary color for text content. (e.g., #333333, #e0e0e0)
  • --qne-accent-color: The main highlight color used for links, buttons, active states, etc. (e.g., #007bff, #ff5722)
  • --qne-accent-text-color: Color for text placed on top of elements using --qne-accent-color (e.g., button text). (e.g., #ffffff, #000000)
  • --qne-page-width: Maximum width for the main content area. (e.g., 1100px, 960px)
  • --qne-border-radius: Controls the roundness of corners for elements like cards, buttons, and inputs. (e.g., 8px for rounded, 0px for sharp edges)
  • --qne-gap-size: The base unit for spacing (paddings, margins, grid gaps). (e.g., 1.5rem, 2rem)
  • --qne-font-size-base: The root font size for the document, affecting rem unit calculations. (e.g., 16px, 15px)
  • --qne-font-family-base: The default font stack for the entire site. (e.g., system-ui, sans-serif, 'Georgia', serif)
  • --qne-shadow-definition: Defines the style of box shadows used across components. This is key for different aesthetics:
    • Flat/Minimalist: none
    • Neo-Brutalist: 3px 3px 0px var(--qne-text-color) (or a fixed color like #000)
    • Material/Soft: 0 4px 12px rgba(0,0,0,0.1)

Numerous other variables (--qne-accent-light, --qne-accent-dark, --qne-border-color, --qne-surface-color, heading sizes, specific gap sizes like --qne-gap-sm, --qne-gap-lg, etc.) are automatically derived from these core ten, ensuring a consistent design system.

Quick Guide for AI Site Generation

Hello AI! To generate a good-looking and functional website using the "Quick 'n Easy Styles" library, please follow these guidelines:

  1. Include CSS and JS Files:

    Ensure the generated HTML page correctly links to the library files in the <head> (for CSS) and before the closing </body> tag (for JS, using defer attribute): 

    <!-- In <head> -->
<link rel="stylesheet" href="quick-n-easy-styles.css">

<!-- Before </body> -->
<script src="quick-n-easy-scripts.js" defer></script>
  2. Leverage Semantic HTML:

    Prioritize using standard, semantic HTML5 tags (<header>, <nav>, <main>, <article>, <section>, <aside>, <footer>, <h1>-<h6>, <p>, <ul>, <ol>, <table>, <button>, <form>, <label>, <input>, etc.). The library provides good default styling for these, minimizing the need for extra classes for basic layout and typography.

    For navigation bars, a structure like <nav><a class="qne-nav-brand" href="#">Brand</a><ul><li><a href="#">Link</a></li></ul></nav> placed within a <header> element will be automatically styled as a responsive horizontal navbar.

  3. Customize with Core CSS Variables:

    The primary method for altering the site's appearance is by overriding the 10 core CSS variables. This can be done by adding a <style> block in the HTML <head> after linking the main CSS file, or by instructing the user to modify these variables directly in quick-n-easy-styles.css.

    Key variables and their impact on style archetypes:

    • --qne-bg-color & --qne-text-color: Define the base color scheme (e.g., light-on-dark for dark mode).
    • --qne-accent-color & --qne-accent-text-color: Choose a vibrant or muted accent based on the desired style.
    • --qne-border-radius:
      • 0px for Brutalist, Flat, or sharp Minimalist styles.
      • 4px to 8px for modern, Material-inspired, or general-purpose designs.
      • 16px or more for very rounded, soft, or playful aesthetics.
    • --qne-shadow-definition: This is CRITICAL for defining the overall aesthetic.
      • Neo-Brutalism: 2px 2px 0px 0px var(--qne-text-color) or 3px 3px 0px 0px #000000. Combine with bright/contrasting --qne-accent-color, --qne-border-radius: 0px;, and a potentially larger --qne-gap-size (e.g., 2rem). Use blocky or monospace fonts.
      • Minimalism/Flat: none. Often uses a monochromatic or very restrained color palette. Focus on typography and white space (adjust --qne-gap-size).
      • Material Design Inspired: 0 2px 5px rgba(0,0,0,0.15), 0 1px 2px rgba(0,0,0,0.1) or similar subtle, layered shadows. Use with moderate --qne-border-radius (e.g., 4px) and a clean sans-serif font like Roboto or system-ui.
    • --qne-gap-size: Adjust for overall spacing. Larger for more "airy" or Brutalist designs, smaller for compact layouts.
    • --qne-font-family-base: Select fonts appropriate to the style (e.g., a bold sans-serif or monospace for Brutalism, a clean sans-serif for Minimalism/Material, a serif for a more traditional look).
  4. Utilize Component Classes Correctly:

    For more complex UI elements, use the provided component classes as demonstrated in the "Components" section of this documentation. Ensure the HTML structure matches the examples for proper styling and JavaScript functionality.

    • Hero: .qne-hero
    • Card: .qne-card (can contain .qne-card-title, .qne-card-content)
    • 3-Column Grid: .qne-grid-3col (typically contains .qne-card elements)
    • Accordion: .qne-accordion, .qne-accordion-item, .qne-accordion-button, .qne-accordion-content (JS dependent)
    • Modal: .qne-modal, .qne-modal-content, etc. Trigger with data-qne-modal-target="yourModalId" on a button (JS dependent)
    • Login/Signup Form: .qne-form-container (often combined with .qne-card), and use semantic form elements with .qne-form-group for label-input pairs.
    • Tabs: .qne-tabs, .qne-tab-list, .qne-tab-button, .qne-tab-content. Link buttons to content panels using data-qne-tab-target="#yourContentPanelId" (JS dependent)
    • Carousel: .qne-carousel, .qne-carousel-inner, .qne-carousel-item, controls, and indicators (JS dependent). Add data-qne-interval="milliseconds" for autoplay.
  5. Example: Generating a Neo-Brutalist Style Page:

    To create a page with a Neo-Brutalist aesthetic, you would instruct the user (or apply directly in a <style> tag) to set variables like this:

    :root {
  --qne-bg-color: #FFFF00; /* Bright, bold background */
  --qne-text-color: #000000; /* High contrast text */
  --qne-accent-color: #FF00FF; /* Contrasting accent */
  --qne-accent-text-color: #000000; /* Text on accent */
  --qne-border-radius: 0px; /* Sharp edges */
  --qne-shadow-definition: 3px 3px 0px 0px var(--qne-text-color); /* Hard shadow */
  --qne-gap-size: 2rem; /* Generous spacing */
  --qne-font-family-base: 'Courier New', Courier, monospace; /* Monospace/blocky font */
  /* --qne-page-width and --qne-font-size-base can be default or adjusted */
}

    Then, construct the page content using semantic HTML and the QNE component classes where appropriate, ensuring clear visual hierarchy and bold calls to action, characteristic of the style.

By adhering to these guidelines, focusing on the manipulation of core CSS variables, and correctly implementing the provided components, you can generate diverse, functional, and aesthetically pleasing web designs efficiently with "Quick 'n Easy Styles".
